# 외부 점검용 공개 데모

운영 Supabase·실사용자 데이터와 **완전히 분리된** 샘플 화면만 보여줄 때 사용합니다.

## 동작 원리

- `ENABLE_DEMO_LOGIN=true` 이고, **문서에 안내된 데모 전용 이메일·비밀번호**로 로그인하면 **브라우저 쿠키만**으로 세션이 열립니다.
- 이 경로는 **Supabase Auth·DB를 호출하지 않습니다.** (`src/lib/data.ts`의 `getDataContext()`가 `demo` 모드가 되어 **코드에 포함된 샘플 데이터**만 사용합니다.)
- 일반 운영 계정은 기존과 같이 Supabase 이메일/비밀번호로 로그인하며, **RLS·데이터 분리는 그대로**입니다.

## 켜는 방법 (예: Vercel Production)

1. **Environment Variables**에 다음을 설정합니다.
   - `ENABLE_DEMO_LOGIN` = `true`
   - `DEMO_LOGIN_EMAIL` = 데모 전용 주소 (예: `demo-review@yourdomain.com`)  
     - **이 이메일로 Supabase에 실제 사용자를 만들지 마세요.** 데모 로그인이 우선하면 해당 주소는 Supabase 로그인에 쓰이지 않습니다.
   - `DEMO_LOGIN_PASSWORD` = **16자 이상** 강한 비밀번호 (임의 생성 권장)
2. **Save** 후 **Redeploy**.

## 끄는 방법

- `ENABLE_DEMO_LOGIN`을 삭제하거나 `false` / `0`으로 두고 **Redeploy**합니다.
- 이미 발급된 데모 비밀번호는 유출 시 **즉시 변경**하세요.

## 점검자(제3자) 안내 문구 예시

- URL: (배포 주소)
- 로그인: `DEMO_LOGIN_EMAIL`에 설정한 주소
- 비밀번호: **별도 안전 채널**으로만 전달 (페이지·README에 비밀번호를 적지 마세요)
- 화면 상단 **「테스트용 데모」** 배너가 보이면 샘플 데이터 모드가 맞습니다.

## 주의사항

| 항목 | 설명 |
|------|------|
| 운영 데이터 | 데모 세션에서는 **DB에 접속하지 않음** → 운영 행 노출 없음 |
| 항상 켜두기 | 외부 시연·점검이 끝나면 **반드시 끔** |
| 비밀번호 | 프로덕션에서 16자 미만이면 데모 로그인이 **거부**됩니다 |
| service_role | 클라이언트·`NEXT_PUBLIC_*`에 넣지 마세요 (기존과 동일) |

## 로컬 개발

- Supabase 없이 개발할 때는 기존과 같이 데모가 열릴 수 있습니다.  
- 프로덕션과 동일한 제약을 맞추려면 `NODE_ENV=production` 빌드와 함께 위 변수로 검증하세요.

## 샘플 데이터 위치

- 데모 UI 데이터: `src/lib/demo-data.ts` (시드 SQL과 별개이며, 데모 모드에서만 사용)

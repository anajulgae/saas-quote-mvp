## Vercel 배포 직후 — 최우선 5개 (먼저 끝내기)

- [ ] **① 로그인** — 프로덕션 URL → 계정으로 로그인 → 대시보드 진입  
- [ ] **② 데모 차단** — 로그인 화면에 데모 계정 안내 **없음** · 데모 이메일/비번으로 로그인 **불가** (`ENABLE_DEMO_LOGIN` 미설정 또는 `false`)  
- [ ] **③ 재로그인** — 로그아웃 → 다시 로그인 → 대시보드 정상  
- [ ] **④ 문의 1건** — 문의 **생성** → 목록에 표시되는지 확인  
- [ ] **⑤ 설정 저장** — 설정 변경 → **저장** → **새로고침** 후 값 유지 · **플랜 컬럼 미적용** 노란 박스가 없는지 확인 (`0004_user_plan.sql`)

## 전체 체크리스트 (우선순위 순 · 체크박스)

### 인증·보안·메시지

- [ ] **1. 로그인 성공** — 프로덕션 URL → 이메일/비밀번호로 로그인 → 대시보드 진입
- [ ] **1b. 회원가입**(선택) — `/signup` → 인증 메일 → 로그인까지 한 번 통과
- [ ] **11. 프로덕션에서 demo login 비활성** — 로그인 페이지에 데모 모드 안내 블록 없음 · 데모 이메일/비번으로 시도 시 로그인 불가
- [ ] **2. 로그아웃 후 재로그인** — 헤더 등에서 로그아웃 → `/login` → 동일 계정으로 재로그인
- [ ] **12. 오류 메시지 노출** — 틀린 비밀번호로 로그인 시 **한글 안내** 표시
- [ ] **13. 비밀번호 재설정** — `/forgot-password` → 메일 → `/reset-password` → 새 비밀번호 → 재로그인
- [ ] **10. 권한 분리** — 다른 계정 B로 로그인 → 계정 A 데이터 **비노출**

### 핵심 업무 흐름

- [ ] **3. 문의 생성/수정** — 문의 등록 → 목록 반영 → 수정 저장
- [ ] **3b. AI 문의 구조화**(선택) — `OPENAI_API_KEY` 설정 시 **AI로 필드 채우기** 동작
- [ ] **4. 견적 생성/수정/상태변경** — 견적 생성 → 수정 → 상태 변경
- [ ] **4b. AI 견적 초안**(선택) — 초안 도우미에서 **AI로 초안 생성** → 견적 폼 반영
- [ ] **4c. 견적 메일** — `RESEND_API_KEY` 설정 시 **보내기**로 테스트 수신 확인
- [ ] **5. 청구 생성/수정/결제상태변경** — 청구 생성 → 수정 → 결제 상태 변경
- [ ] **6. 리마인드 기록** — 리마인드 저장 → 이력 표시 · (선택) AI 문구

### 데이터·설정·탐색

- [ ] **7. 고객 상세 타임라인** — 고객 상세에서 타임라인 기록 확인
- [ ] **8. 설정 저장** — 사업자·템플릿 저장 후 새로고침 유지
- [ ] **9. 검색/필터** — 문의·고객·견적·청구 중 목록 좁히기 동작

### 랜딩·플랜

- [ ] **14. `/billing`** — 요금·업그레이드 안내 페이지 열림 · Pro CTA → 가입 흐름 연결

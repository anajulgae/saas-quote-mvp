# 운영자용 — 배포 직후 10분 스모크 체크리스트

**대상**: Vercel + Supabase 비공개 베타 배포 직후  
**목표**: 약 10분 안에 핵심 동작·보안·메시지를 확인합니다.  
**실행 순서(Runbook)**: [deploy-runbook.md](./deploy-runbook.md) · **상세 배포**: [deployment.md](./deployment.md) · **RC 체크**: deployment §8  
**실패 시**: Supabase 마이그레이션(`0001`→`0002`→`0003`) · Vercel 환경변수 · Auth Site URL·Redirect URL을 먼저 확인하세요.

---

## Vercel 배포 직후 — 최우선 5개 (먼저 끝내기)

- [ ] **① 로그인** — 프로덕션 URL 접속 → 베타 계정으로 로그인 → 대시보드 진입  
- [ ] **② 데모 차단** — 로그인 화면에 데모 계정 안내 **없음** · 데모 이메일/비번으로 로그인 **불가** (`ENABLE_DEMO_LOGIN` 미설정 또는 `false`)  
- [ ] **③ 재로그인** — 로그아웃 → 다시 로그인 → 대시보드 정상  
- [ ] **④ 문의 1건** — 문의 **생성** → 목록에 표시되는지 확인  
- [ ] **⑤ 설정 저장** — 설정 변경 → **저장** → **새로고침** 후 값 유지

## 전체 체크리스트 (우선순위 순 · 체크박스)

아래는 위 5개 이후·병행으로 진행해도 됩니다. **위에서 막히면 아래는 잠시 보류**하세요.

### 인증·보안·메시지

- [ ] **1. 로그인 성공** — 프로덕션 URL → 이메일/비밀번호로 로그인 → 대시보드 진입
- [ ] **11. 프로덕션에서 demo login 비활성** — 로그인 페이지에 데모 모드 안내 블록 없음 · 데모 이메일/비번으로 시도 시 로그인 불가(또는 설정 안내 화면만)
- [ ] **2. 로그아웃 후 재로그인** — 헤더 등에서 로그아웃 → `/login` → 동일 계정으로 재로그인 → 정상 진입
- [ ] **12. 오류 메시지 노출** — 의도적으로 **틀린 비밀번호**로 로그인 시도 → 화면에 **한글 안내 문구**가 뜨는지 확인(빈 화면·무응답 아님)
- [ ] **10. 권한 분리** — **다른 베타 계정 B**로 로그인(또는 시크릿 창) → 계정 A에서 만든 문의/견적이 **보이지 않음**을 확인

### 핵심 업무 흐름

- [ ] **3. 문의 생성/수정** — 문의 **등록** → 목록 반영 → 동일 건 **수정 저장** → 내용 반영
- [ ] **4. 견적 생성/수정/상태변경** — 견적 **생성** → **수정** 저장 → 드롭다운 등으로 **상태 변경** → 카드에 반영
- [ ] **5. 청구 생성/수정/결제상태변경** — 청구 **생성** → **수정** 저장 → **결제 상태** 변경 → 카드에 반영
- [ ] **6. 리마인드 기록 생성** — 청구 카드에서 **리마인드** → 내용 저장 → 해당 청구의 리마인드 이력에 표시

### 데이터·설정·탐색

- [ ] **7. 고객 상세 타임라인** — 위에서 만든 문의/견적/청구가 연결된 **고객 상세**로 이동 → **타임라인**(활동)에 기록이 보이는지 확인
- [ ] **8. 설정 저장** — **설정**에서 사업자 정보 또는 템플릿 수정 → **저장** → 페이지 새로고침 후 유지 확인
- [ ] **9. 검색/필터** — **문의** 검색창 동작 · **고객** 검색 · **견적** 상태 필터 · **청구** 빠른 필터(미수금/연체 등) 중 1개 이상으로 목록이 줄어드는지 확인
